# PRD 추가 섹션: 비채점 상호작용과 친밀도

## 기능 A. 놀아주기 — 데이터의 바다 산책

에아몬이 데이터의 바다에서 주워온 것을 반에 보여주는 비채점 상호작용이다. 채점, 경험치, 선악 게이지에는 영향을 주지 않고 정 주기와 다음 학습 떡밥 역할을 한다.

- 메인 화면에 `놀아주기` 버튼을 둔다.
- 누르면 에아몬이 "나 오늘 바다에서 이런 거 봤어!"라고 말하고 랜덤 발견물 1개를 보여준다.
- 발견물 풀은 `good`, `weird`, `plain` 세 종류다.
- `weird` 발견물은 `linked_episode_code`를 가질 수 있으며, 이후 해당 V-red 에피소드가 우선 노출된다.
- 산책 1회마다 친밀도 `+1`.

데이터:

```text
walk_items(id, type, content_text, image_url, linked_episode_code)
walk_logs(id, aemon_id, item_id, created_at)
```

## 기능 B. 오염 데이터 청소 — 정화

에아몬이 바다를 헤엄치다 나쁜 데이터 조각을 묻혀오는 비채점 상호작용이다. 미디어 리터러시 소재를 가볍게 반복 노출한다.

- 오염 상태가 되면 에아몬 외형에 얼룩/노이즈와 알림점을 표시한다.
- 메인 화면에 `정화하기` 버튼을 노출한다.
- 정화 화면에서는 오염물의 정체와 한 줄 설명을 보여준다.
- 사용자가 청소 버튼을 탭해 닦아내면 에아몬이 깨끗해지고 감사 대사를 말한다.
- 정화 1회마다 친밀도 `+1`.
- 방치해도 페널티는 없고, 외형의 오염 표시만 유지된다.

데이터:

```text
pollution_items(id, axis, label, one_liner, linked_episode_code)
aemons.is_polluted
aemons.pollution_item_id
clean_logs(id, aemon_id, item_id, created_at)
```

## 공통. 친밀도 축

친밀도는 선악 게이지와 완전히 분리된 제3의 축이다. 산책과 정화 같은 비채점 상호작용으로만 쌓인다.

- 진화 형태와 졸업 엔딩의 선악에는 영향을 주지 않는다.
- idle 혼잣말을 더 친근하게 만든다.
- 졸업 대사 깊이에 영향을 준다.
- 선악 엔딩과 친밀도는 2축으로 조합된다.

데이터:

```text
aemons.intimacy int default 0
```

## 혼잣말: 직전 에피소드 반영

메인 idle 혼잣말은 마지막으로 완료한 에피소드가 있을 경우 해당 내용을 우선 반영한다. 없거나 일정 확률로 폴백할 때는 단계×성향 매트릭스를 사용한다.

예:

```text
V-02 이후: "어제 너희가 똑같이 갚으면 나도 똑같아진다고 했지. 계속 생각 중이야."
```
